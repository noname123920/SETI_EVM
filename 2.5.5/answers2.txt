

ЧАСТЬ 1. Настройка основных параметров коммутаторов

Вопрос: Как можно проверить правильность настройки паролей?
Ответ: Нужно выйти из привилегированного режима (exit), затем нажать Enter 
       и ввести пароль консоли для входа в пользовательский режим. После 
       этого ввести команду enable и ввести пароль привилегированного режима. 
       Также можно посмотреть файл running-config (show running-config), 
       где будут отображаться настроенные пароли.

Вопрос: Какую команду необходимо выполнить для сохранения конфигурации в NVRAM?
Ответ: copy running-config startup-config (или сокращенно copy run start)

ЧАСТЬ 2. Настройка ПК

Вопрос: Удалось ли выполнить ping? Дайте пояснение.
Ответ: Нет, ping не должен пройти (будет "Request timed out" или 0% успеха).
       Пояснение: Коммутатор является устройством уровня 2 и не имеет 
       настроенного IP-адреса по умолчанию. Он не отвечает на ICMP-запросы,
       пока не будет настроен интерфейс управления VLAN 1.

ЧАСТЬ 3. Настройка интерфейса управления коммутатором

Вопрос: Для чего тогда нужно настраивать IP-адреса?
Ответ: IP-адрес настраивается на коммутаторе для удаленного управления 
       устройством (по Telnet/SSH), для мониторинга сети, а также для 
       возможности отправлять и получать служебные протоколы (например, 
       обновления по сети или использование протоколов маршрутизации, если 
       коммутатор работает на уровне 3). Сам по себе коммутатор работает 
       и без IP-адреса, но управлять им можно только через консоль.

Вопрос: Зачем вы вводите команду no shutdown?
Ответ: Команда no shutdown включает интерфейс. По умолчанию все интерфейсы 
       на коммутаторе (включая виртуальный интерфейс VLAN 1) находятся в 
       административно выключенном состоянии (administratively down). 
       Команда no shutdown переводит интерфейс в активное состояние (up).

Вопрос: Какая команда сохраняет файл конфигурации из RAM в NVRAM?
Ответ: copy running-config startup-config (или copy run start)
