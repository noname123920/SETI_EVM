
ЧАСТЬ 1. Проверка конфигурации по умолчанию

1. Сколько у коммутатора интерфейсов Fast Ethernet?
   Ответ: 24

2. Сколько у коммутатора интерфейсов Gigabit Ethernet?
   Ответ: 2

3. Каков диапазон значений, отображаемых в vty-линиях?
   Ответ: 0 4 (означает 5 линий для удаленного доступа)

4. Какая команда отображает текущее содержимое NVRAM?
   Ответ: show startup-config

5. Почему коммутатор отвечает "startup-config is not present"?
   Ответ: Конфигурация еще ни разу не сохранялась в NVRAM. 
          Коммутатор работает с настройками по умолчанию из RAM.

ЧАСТЬ 2. Настройка основных параметров

6. Для чего нужна команда login?
   Ответ: Включает запрос пароля. Без нее система не спросит пароль,
          даже если он установлен командой password.

7. Что отображается в качестве пароля enable secret?
   Ответ: Зашифрованная строка (хеш), например $1$mERr$4xrJ4/SEk...

8. Почему enable secret отображается не как заданный текст?
   Ответ: Используется необратимое хеширование (MD5) вместо простого
          шифрования. Это более надежная защита.

9. Будут ли новые пароли шифроваться после service password-encryption?
   Ответ: Да. Команда включает шифрование для всех текущих и будущих
          паролей (кроме enable secret).

ЧАСТЬ 3. Настройка баннера MOTD

10. Когда будет отображаться баннер?
    Ответ: До запроса на вход в систему (до появления "User Access 
           Verification"), сразу при подключении.

11. Зачем нужен баннер MOTD?
    Ответ: Для юридического предупреждения о том, что доступ разрешен
           только авторизованным пользователям.

ЧАСТЬ 4. Сохранение конфигурации

12. Самая короткая версия copy running-config startup-config?
    Ответ: copy run start

13. Какая команда отображает содержимое NVRAM?
    Ответ: show startup-config (или show start)

14. Все ли изменения записаны в файл?
    Ответ: Да, если появилось сообщение [OK]. Можно проверить командой
           show startup-config.
